# 朗读功能说明

## 功能概述
本站使用浏览器内置的 Web Speech API 提供文本朗读功能，无需额外安装任何插件或应用。

## 离线朗读限制

### 桌面端
- **Chrome/Edge（桌面版）**：✅ 支持离线朗读（使用系统内置语音）
- **Firefox**：⚠️ 部分支持（取决于操作系统）
- **Safari（macOS）**：✅ 支持离线朗读

### 移动端
- **iOS Safari**：❌ **需要网络连接**
  - iOS 的语音合成服务需要联网才能工作
  - 这是 iOS 系统的限制，不是本站的问题
  
- **Android Chrome**：⚠️ **部分支持**
  - 如果设备已下载中文语音包，可以离线使用
  - 否则需要网络连接
  
- **Android 其他浏览器**：⚠️ 取决于浏览器和系统设置

## 解决方案

### 方案 1：使用桌面浏览器（推荐）
在电脑上使用 Chrome、Edge 或 Safari 浏览器，可以完全离线朗读。

### 方案 2：保持网络连接（移动端唯一方案）
在移动端使用时，保持 WiFi 或移动数据连接。

**注意：** Android 下载语音包对 Web 应用无效，因为 Web Speech API 仍然需要联网验证。

## 错误提示说明

- **"离线模式"**：检测到设备离线，但仍会尝试播放
- **"需要网络"**：语音服务需要网络连接
- **"语音不可用"**：浏览器不支持或语音服务未启用
- **"播放失败"**：其他播放错误

## 技术说明

Web Speech API 是浏览器标准 API，其离线能力取决于：
1. **浏览器实现**：移动端浏览器的语音合成服务通常需要联网
2. **操作系统支持**：iOS 完全不支持离线语音合成
3. **网络依赖**：即使设备有语音包，Web API 也可能需要联网验证

**这不是本站可以控制的，而是浏览器和操作系统的限制。**

## 为什么不能通过代码解决？

1. Web Speech API 是浏览器提供的接口，我们无法控制其实现
2. 移动端浏览器（特别是 iOS）的语音服务是云端服务
3. 即使尝试选择"本地语音"，在移动端也不起作用
4. 唯一的替代方案是预录音频文件，但会导致：
   - 文件体积巨大（每篇可能几十 MB）
   - 无法调整语速
   - 无法实时生成

## 建议

如果您经常需要离线使用朗读功能：
1. **桌面端**：使用 Chrome/Edge/Safari 浏览器（完全支持离线）
2. **移动端**：保持网络连接（这是唯一方案）
3. **iOS 用户**：必须联网才能使用朗读功能
