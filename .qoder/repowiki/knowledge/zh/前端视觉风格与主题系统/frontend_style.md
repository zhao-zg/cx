该平台采用**纯 CSS 变量（CSS Custom Properties）驱动的主题系统**，配合**静态生成器（Python/Jinja2）**输出的 HTML 结构，实现了一套轻量、离线优先且高度一致的视觉风格。整体设计偏向**阅读密集型应用**，强调内容的清晰度、层级感以及多环境（冷色/暖色/夜间）下的舒适性。

### 1. 核心样式架构
*   **单一全局样式表**：所有页面共用 `css/style.css`（约 1100+ 行），由 Python 生成器在构建时复制到输出目录。这种“单文件”策略极大简化了 PWA 和离线环境的资源加载与缓存管理。
*   **CSS 变量令牌（Design Tokens）**：通过 `:root` 定义基础常量（如 `--brand`, `--radius-sm`），并通过 `[data-theme="..."]` 属性选择器覆盖语义化变量（如 `--surface`, `--text`, `--header-bg`）。组件样式仅引用语义变量，不硬编码颜色，实现了主题切换的零 JS 渲染开销。
*   **响应式布局**：采用**移动端优先**策略。基础布局使用 `max-width: 800px` 的居中容器，配合 `env(safe-area-inset-*)` 适配刘海屏。媒体查询（`@media (min-width: 768px)`）主要用于调整内边距、字号和网格间距，确保在大屏设备上拥有更宽松的呼吸感。

### 2. 主题系统设计
平台内置三种核心主题，通过 `html[data-theme]` 动态切换：
*   **冷色模式（Cool/Default）**：以蓝紫色调（`#667eea`）为主，背景为极淡的蓝白渐变，营造现代、清爽的阅读氛围。
*   **暖色模式（Warm/Paper）**：模拟纸质书质感。使用奶油色背景（`#FBF7EF`）搭配深棕色文字（`#4A3525`），品牌色转为大地色系（`#8B6B3A`），显著降低蓝光刺激，适合长时间阅读。
*   **夜间模式（Dark）**：深蓝灰色背景（`#181b21`），配合低对比度的灰蓝色文字，避免纯黑背景带来的视觉疲劳，并针对进度条、按钮等高亮元素做了专门的深色适配。

### 3. 排版与内容呈现
*   **层级分明的纲目系统**：针对特会信息的结构化特点，定义了 `.section-level1` 到 `.level-5` 五级样式。通过**缩进（margin-left/padding-left）**、**负文本缩进（text-indent）**以及**字重/字号**的细微变化，在视觉上清晰呈现“大纲-中纲-小纲”的嵌套关系。
*   **经文区块专用样式**：`.scripture-block` 等类使用左侧边框（`border-left: 3px solid var(--brand)`）和浅色背景进行区分，确保经文引用在正文中既醒目又不突兀。
*   **字体栈**：优先使用系统字体（`-apple-system, BlinkMacSystemFont, PingFang SC, Microsoft YaHei`），确保在各平台下的原生渲染效果和加载速度。

### 4. 交互与微反馈
*   **统一的触控反馈**：所有可点击元素（按钮、导航链接）均设置了 `min-height: 44px` 以满足无障碍触控标准，并配有 `:active` 态的缩放（`scale(0.95)`）或背景色变化反馈。
*   **浮动面板与遮罩**：设置面板、经文弹框、高亮菜单均采用固定定位（`position: fixed`）配合半透明遮罩。在移动端通过 `touch-action` 和 JS 辅助逻辑防止滚动穿透，并提供从底部滑入或中心弹出的动画过渡。
*   **功能型 UI 组件**：包括底部的朗读控制栏（含进度条、语速选择）、顶部的阅读进度条（`cx-reading-progress`）以及图片查看器的全屏手势支持，均通过 CSS 变量保持与当前主题的高度融合。

### 5. 开发约定
*   **禁止硬编码颜色**：在编写新样式时，必须使用 `var(--text)`, `var(--surface)` 等语义变量，严禁直接使用 `#fff` 或 `#000`（除非是特定主题的预览卡片）。
*   **状态类命名**：采用 `cx-` 前缀（如 `cx-highlight`, `cx-search-target`）标识由 JS 动态控制的功能性样式，避免与静态布局类冲突。
*   **离线兼容性**：样式设计不依赖外部 CDN 字体或图标库，所有图标均使用内联 SVG 或 Unicode 字符，确保在无网络环境下 UI 完整可用。
