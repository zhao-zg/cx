## 修复 theme-toggle.js 启动 CORS 控制台噪音
- 现象：本地 `127.0.0.1:5500` 打开页面，控制台刷 3 条 CORS 红字（`cx.zhaozg.cloudns.org` 等 403 + No ACAO）。用户误以为导致"页面放大/显示不全"。
- 根因：`checkServerReachability()` 用默认 `cors` 模式对 `CX_SERVERS.cloudflare` 发跨域 `HEAD`，本地跨域被浏览器拦截并打印 CORS 错误（已被 `.catch` 静默，不影响渲染）。"放大/显示不全"实与 CORS 无关（浏览器缩放或 `localStorage.globalFontSize` 偏大）。
- 关键冲突：`mode:'no-cors'` 与"保留 `r.ok` 语义"互斥——no-cors 返回 opaque 响应，`r.ok` 恒为 false、`r.status` 恒为 0；若保留 `resolve(r.ok)` 则探测永远判不可达（连生产 200 也隐藏按钮）。因此不能简单用 no-cors + r.ok。
- 最终修复（用户要求保留"仅 2xx 才算可达"语义）：`checkServerReachability()` 新增 `_isLocalDevOrigin()`（localhost/127.0.0.1/::1/file://），**本地开发直接跳过跨域探测、判不可达**（与生产"跨域 403 → 不可达"结果一致）；生产（同源）仍走原始 `fetch` + `resolve(r.ok)`，逻辑零改动。本地零 CORS 红字、生产语义不变。
- 注：第一版误用 no-cors+resolve(true)（把 403/404 也算可达），用户指出不该改语义，已回退为该 dev-guard 方案。
- 注意：`output/` 属构建产物，修改会被覆盖；若需长期生效须走 `main.py` 重新生成或提交 src 改动。
